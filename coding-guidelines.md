# Coding & Design Guidelines
- **Tech Stack:** Next.js with pure HTML/CSS. Strictly NO heavy external UI libraries. Keep it lightweight.
- **Layout:** Fully responsive design. The maximum container width should be exactly 1600px.
- **SEO:** High priority on Core Web Vitals, Semantic HTML, structured data (Schema.org for products), and fast load times.
- **Language & Typography:** Arabic (Right-to-Left RTL support is mandatory from the start). Simple, direct phrasing.
- **Terminology Rules (Strict):**
  - ALWAYS use: نكهة, سحبة, كويل, ميش كويل, بودات, سحبات جاهزة, تانك, رقي, ايس, مبسم, قوة, سبيرمنت.
  - NEVER use: السائل الالكتروني, موشة, بطيخ, ملف, خراطيش, أنظمة البود, تبريد, قطعة فم.
  - AVOID filler phrases like: "حسب التوفر" or "كما هو موضح بالصورة".
- **Content Output:** Product descriptions should be kept concise, aiming for around 160 characters for optimal UI/SEO display.
